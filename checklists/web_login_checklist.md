## Позитивные сценарии (валидный вход):
1. Вводим валидные данные (standard_user/secret_sauce) - вход успешный
2. Вводим валидные данные (problem_user/secret_sauce) - вход успешный
3. Вводим валидные данные (performance_glitch_user/secret_sauce) - вход успешный

## Негативные проверки (не валидный вход)
1. Оба поля пустые - ошибка, вход не выполнен
2. Пароль пустой, Имя заполнено - ошибка, вход не выполнен
3. Имя пустое, пароль заполнен - ошибка, вход не выполнен
4. Имя цифры - ошибка, вход не выполнен
5. Имя спецсимволы - ошибка, вход не выполнен
6. Имя более 200 знаков - ошибка, вход не выполнен
7. Пароль более 200 знаков - ошибка, вход не выполнен
8. Неверный пароль при валидном Имени - ошибка, вход не выполнен
9. Неверное Имя при валидном пароле - ошибка, вход не выполнен
10. Оба поля не валидные - ошибка, вход не выполнен
11. Пробелы в конце Имени - ошибка, вход не выполнен
12. Пробелы в начале Имени - ошибка, вход не выполнен
13. Пробелы в конце Пароля - ошибка, вход не выполнен
14. Пробелы в начале Пароля - ошибка, вход не выполнен
15. Имя в другом регистре (PERFORMANCE_GLITCH_USER) - ошибка, вход не выполнен
16. Пароль в другом регистре (SECRET_SAUCE) - ошибка, вход не выполнен
17. Имя с внутренними пробелами (pro blem_user) - ошибка, вход не выполнен

## UX/Валидации
1. При ошибке входа, пользователь остаётся на странице логина
2. Сообщение об ошибке видно и хорошо читается
3. Ошибка не перекрывает кнопку и поля
4. Сообщение об ошибке можно закрыть нажатием на крестик
5. При ошибке ввода данных можно ввести повторно, без перезагрузки окна

## Клавиатура/Удобство
1. Клавишей Tab можно перейти с поля Имя на поле Пароль
2. Вход можно выполнить клавишей Enter
3. Текст в полях можно выделять и удалять
4. Можно очищать поля (Ctrl+A - Delete)
5. Можно вставлять в поля (Ctrl+V)

## Мини-безопасность/Logout
1. Пароль скрыт точками
2. После logout при нажатии Back нельзя попасть в защищённую зону, Система требует авторизацию
